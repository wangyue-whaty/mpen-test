一、 整体说明：

基于maven实现的多环境动态加载配置文件。

二、 环境说明（所有环境配置文件均在classpath:profile目录下）：

1、 dev：本地开发环境；

2、 test：百度云测试环境（ip:180.76.134.187）；

3、 prop：线上环境（不包含定时任务功能）（ip:125.208.27.140）；

4、 timer：线上定时任务处理环境（包含所有功能，但只执行定时任务，不对外提供服务）（ip:125.208.27.140）。

三、 操作说明：

  • 1、本地开发(dev环境)：

    (1) 本地开发通过直接运行MpenApiApplication中Main方法启动服务，默认加载dev环境配置文件，可通过修改pom.xml文件中activeByDefault属性更改默认加载的配置文件；

    (2) 通过maven命令 “mvn spring-boot:run -P ‘环境代号’”启动服务，未携带环境参数默认加载dev环境配置文件；

  • 2、linux环境部署(test,prop,timer环境)：

    (1) 本地通过maven命令“mvn clean package -P ‘环境代号’”动态加载不同环境配置文件打成jar包，未携带环境参数默认加载dev环境配置文件；

    (2) 通过执行命令“java -jar 'jar包'”启动服务。

四、 备注：

在部署jar包过程中，如果需要自定义application.properties配置文件，可在jar包所在同级目录下，放置自定义application.properties配置文件，服务启动时会优先加载同级目录下的配置文件。
